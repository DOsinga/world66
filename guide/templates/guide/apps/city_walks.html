City Walks

Turn World66 POIs into guided walking routes. Street-level navigation, offline maps, curated neighbourhood tours. The data exists — it needs a front-end.

This is a concept, not a finished app

City Walks doesn't exist yet. This page describes what it could be. If you're a developer or designer who wants to build it on the World66 platform, we'd love to hear from you.

The opportunity

20,000 geocoded POIs. Already in a folder. Waiting for a walking app.

World66 contains tens of thousands of points of interest — museums, cafés, viewpoints, street art, markets — each with latitude, longitude, description, and category. That's exactly the data a city walk app needs.

The guide already has curated neighbourhood sections. Each one is a natural walking route: pick five to eight POIs, add turn-by-turn context, render it on a map.

Because the content is CC BY-SA and the code is open source, you could fork World66, read from the same content/ folder, and ship a walk app in a weekend.

Ready-made content
Every city in World66 has neighbourhood sections with curated POIs. Convert each section to a walk: start, stop, waypoints, and descriptions already written.
Offline-first
Walks are just markdown + GeoJSON. Bundle a city at install time. No cell signal needed once the route is on the device.
AI-generated routes
A Claude Code agent could read a city's POIs, cluster them geographically, and propose walk routes — then write them back into the content folder for human review.
Same licence, same stack
CC BY-SA content, Python/Django backend, Leaflet maps. The same technologies as World66 and TimeSpace — no new infrastructure to learn.
