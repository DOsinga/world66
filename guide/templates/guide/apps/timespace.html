TimeSpace

Local events, connected to the places you already know.

TimeSpace sits on top of the World66 POI database. Every concert venue, museum, and theatre is already geocoded in World66 — TimeSpace adds a time dimension: what's happening there this week?

AI agents run weekly, fetching schedules from venue websites. Events are stored as markdown files and committed via pull requests. You review; the agent fetches.

Currently covering Amsterdam, Rotterdam, Utrecht, The Hague, Amersfoort, Gouda — with more cities easy to add by dropping a source file.

See what's on View on GitHub
Agentic event fetching
Source files define a URL, venue POI, and refresh interval. An AI agent reads each source, scrapes events, and submits a PR — no manual copy-pasting.
Events as markdown
Each event is a YYYY-MM-DD-slug.md file with YAML frontmatter — date, category, venue POI path, URL.
Standalone or embedded
Runs inside World66 at /timespace/, or as a fully standalone Django app.
Add a city in minutes
Drop a source file pointing at a venue's agenda URL. The agent does the rest.
