Why World66

In the age of AI, human knowledge matters more — not less.

Every LLM was trained on the same web. They all produce the same advice. The only way to be different is to know something real. World66 is built on that difference.

🧠
You bring the knowledge

You've been there. You know which neighbourhood to stay in, which market to avoid, and what's worth the detour. That can't be generated.

🤖
AI does the heavy lifting

Claude Code acts as your co-author: it researches facts, formats content, finds opening hours, and sends PRs — so you can focus on what only you know.

🌍
Together we build something new

The result is a living guide: constantly updated, deeply human, and built by people who actually care about the places they write about.

Pure AI travel content

  • Generic advice synthesised from millions of identical sources
  • Confident hallucinations about places that have closed or never existed
  • No point of view — no one is responsible for being wrong
  • Optimised for search rankings, not for the traveller reading it

World66 — human + AI

  • Local and traveller knowledge that isn't anywhere else online
  • Verified by people who have walked the streets, not scraped them
  • Opinionated — someone stands behind every recommendation
  • AI handles research and formatting; humans handle truth and taste
Agentic AI

One person with an AI agent can do more than a team could in 2010.

We use Claude Code as an agentic co-author. It can browse sources, write descriptions, geocode venues, and submit pull requests. You point it at a city; it does the groundwork.

Research at scale

The agent finds event listings, exhibition schedules, venue details, and local sources — across dozens of cities simultaneously.

Consistent formatting

Every page follows the same structure: YAML frontmatter, markdown body, standardised categories. The agent enforces it so you don't have to.

You stay in control

Everything goes through Git and pull requests. You review what the agent produced, edit anything that feels wrong, and merge when it's right.

Open content · Open source

Everything here is free. Forever.

World66 was one of the first open-content travel guides on the web, launched in 1998 under a Creative Commons licence. That hasn't changed. Every word you write here is free for anyone to read, share, and build on.

The code is open source too. There are no investors to satisfy, no paywall to build toward, no acquisition strategy. The only goal is a great travel guide that stays independent.

When you contribute, your work joins a commons that anyone — a student in Nairobi, a developer in Seoul, a traveller in Peru — can use without asking permission or paying a fee.

How to contribute

From zero to your first contribution in an afternoon.

The full workflow runs through Git and GitHub. You write markdown, AI helps where useful, and everything is reviewed before it goes live.

Fork and clone the repo

Head to github.com/DOsinga/world66, fork it, and clone your fork locally. All content lives in the content/ folder as plain markdown files.

Pick a place you know

Find the city, neighbourhood, or point of interest in content/. If it doesn't exist yet, create it. The folder structure mirrors the world: content/europe/netherlands/amsterdam/

Read CLAUDE.md and STYLE.md

These two files explain exactly how pages should be structured and how to write in the World66 voice. Read them once and you'll know what good looks like.

Write or improve the content

Edit the markdown directly, or open the repo in Claude Code and ask it to help. It knows the style guide and can draft, research, and format for you.

Run the site locally to preview

Install dependencies with pip install -r requirements.txt, then run python manage.py runserver 8066. Visit localhost:8066 to see exactly how your changes will look.

Submit a pull request

Push your branch and open a PR on GitHub. Describe what you changed and why. We review, give feedback if needed, and merge. Your contribution goes live on the next deploy.
